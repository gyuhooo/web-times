# Timecode Display for OBS

現在のタイムコード（時刻）をリアルタイム表示するシンプルな1ファイルWebページです。
文字色・サイズ・フォント・縁取りなどを自由に変更でき、**背景透過**に対応しているので
OBSのブラウザソースに重ねて使えます。

## 使い方

### ローカルで開く
`public/index.html` をブラウザで開くだけです。

### OBSで使う
1. ソース → **ブラウザ** を追加
2. **ローカルファイル** にチェックを入れ、`public/index.html` を指定
   （もしくはCloudflare Pages等で公開したURLを指定）
3. 幅・高さを配信解像度に合わせる（例: 1920 × 1080）
4. 背景設定が「透過」になっていれば、そのまま透けて合成されます

> OBSの「カスタムCSS」欄は **空のままでOK** です。

## 設定パネル

- 画面右上の **歯車アイコン**、または **S キー** で設定パネルを開閉します。
- パネルはOBSのキャプチャには映りません（操作用UIです）。
- 変更内容はブラウザに自動保存されます。

### 変更できる項目
- **モード**: 時計 / タイマー（カウントダウン）/ ストップウォッチ / イベントまでの残り時間
- 表示形式（時:分:秒 / 小数付き〔桁数選択〕/ フレーム付きタイムコード / 時:分）
- フレームレート（タイムコード形式のとき）
- 24/12時間表示、時の先頭ゼロ、日付の表示と書式
- **タイムゾーン**（ワールドクロック。UTC・各都市。小数秒も保持）
- 文字色、サイズ、フォント、太さ、文字間隔、透明度
- 「:」の点滅、画面内の配置（縦横3×3、OBSの位置決め用）
- 小数・フレームの位置（上付き/下付き/上揃え 等）
- エフェクト: グロー / ネオン / グラデーション / アウトライン（中空）/ パルス / レインボー / グリッチ / シェイク（色・速さ調整可）
- 縁取り・影、背景（透過/単色）、文字背後の半透明パネル（視認性UP・既定OFF）
- 透過確認用の市松模様（編集中だけ表示）

### タイマー / ストップウォッチ
- タイマー: 時分秒を設定 → 開始 / 一時停止 / リセット。終了時に点滅・メッセージ・ビープ音（要許可）。
  - **段階アラート**: 残り時間で文字色が 通常 → 注意（黄）→ 警告（赤）に変化（しきい値・色を設定可）。
  - **オーバータイム**: 0:00以降も `+MM:SS` で超過時間をカウントアップ（登壇・LTの時間管理向け）。
  - **プログレスバー**: 残り時間をバーで可視化（色・太さ調整可。段階アラートの色と連動）。
- ストップウォッチ: 開始 / 一時停止 / リセット / ラップ。
- いずれもタブが非アクティブでもズレない絶対時刻方式。
- ショートカット（パネルを閉じた状態）: `Space`=開始/一時停止, `R`=リセット, `L`=ラップ。

### 時刻同期（NTP相当）
ブラウザは本物のNTP（UDP）を使えないため、HTTPで権威ある時刻に補正します。優先順位は
同一オリジンの Cloudflare `/cdn-cgi/trace`（このサイトはCloudflare Pages上なのでCORS不要・高精度）→
`timeapi.io` → 公開 `cloudflare.com/cdn-cgi/trace` → `worldtimeapi.org` → 同一オリジンの `Date` ヘッダ → ローカル時計。
SNTP方式（往復遅延を補正）で最もRTTの小さいソースを採用し、5分ごと＋復帰時に再同期します。
設定パネルに現在の同期元と補正量が表示されます。

## 複数シーンで使い分ける

設定パネルの **「設定付きURLをコピー」** ボタンで、現在の見た目をすべて
クエリパラメータに含んだURLを生成できます。OBSの各ブラウザソースに別々のURLを
指定すれば、シーンごとに色やサイズを変えられます。

例: `index.html?format=HMSf&color=%2300ff66&size=14&shadowOn=1`

## Cloudflare Pages へのデプロイ

静的サイトなのでビルド不要です。配信ディレクトリは `public/` です。

### A. CLIで直接デプロイ（最速）
[APIトークン](https://dash.cloudflare.com/profile/api-tokens)（**Cloudflare Pages: Edit** 権限）と
アカウントIDを用意して、次を実行します。

```bash
export CLOUDFLARE_API_TOKEN=xxxxxxxx
export CLOUDFLARE_ACCOUNT_ID=xxxxxxxx
npx wrangler pages deploy public --project-name=web-times
```

### B. GitHub Actionsで自動デプロイ
`.github/workflows/deploy.yml` を同梱しています。GitHubリポジトリの
**Settings → Secrets and variables → Actions** に以下を登録すると、push時に自動デプロイされます。
- `CLOUDFLARE_API_TOKEN`
- `CLOUDFLARE_ACCOUNT_ID`

### C. ダッシュボードでGit連携
Cloudflareダッシュボード → **Workers & Pages → Create → Pages → Connect to Git** で
このリポジトリを選択。ビルドコマンドは空、**ビルド出力ディレクトリ**を `public` に設定します。
（トークンを共有せずに済む方法です）
